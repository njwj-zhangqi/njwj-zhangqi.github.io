HTTP

HTTP请求的过程

DNS解析

  1. 查看本地缓存是否存在
  2. 本地缓存没有的话查看host文件
  3. host也没有，到最近的DNS服务器查询
  4. 没有会一直到域名根服务器
  5. 全都没有，则报错

HTTP交互四步

  1. 建立TCP连接（三次握手）
  2. 发送请求报文（request）报文组成：请求行、请求头、空行、请求数据
    • 请求行：请求的目标
    • 请求头：客户端的相关信息
    • 空行：请求头的结束
    • 请求数据：
  3. 服务端响应请求（response）
    • 根据GET和POST交给Servlet的doGet和doPost方法来处理
    • 响应报文组成：状态行、响应头、空行、响应数据
      • 状态行：HTTP版本、响应代码、响应描述
      • 响应头：服务器的相关信息、响应数据的信息
      • 空行：响应头的结束
      • 响应数据：HTML文档和图像等，也就是HTML文件
  4. 关闭连接（四次挥手），客户端解析HTML

TCP和UDP

TCP面向连接，UDP发送之前不需要建立连接

TCP可靠，UDP不可靠

TCP通过字节流，UDP通过报文

TCP为什么不两次握手

TCP头部组成中包括：序号（seq），确认号（ack），flags（标记位）

标记位共6个：URG、ACK、PSH、RST、SYN、FIN

  • SYN：建立新连接
  • ACK：确认序号有效
  • FIN：释放一个连接

三次握手的流程：

Client =》 Server：SYN=1，seq=x（客户端建立连接）

Server =》Client：SYN=1,seq=y(服务器建立连接)，ACK=1,ack = seq + 1（确认客户端）

Client =》Server：SYN，seq=x+1，ACK=1，ack=y+1（确认服务端）

服务器发送确认数据包就会建立连接，客户端如果没有相应请求，服务器则会一直等待，白白浪费资源。三次握手则能判断客户端和服务器都已经准备好。

HTTP协议

用于定义Web浏览器和服务器之间通讯格式

Http1.0和Http1.1的区别

HTTP1.0协议中，一个连接只能获得一个Web资源，HTTP1.1协议中，一个连接可以获得多个Web资源

HTTP请求格式

  • 请求行：描述客户的请求方式、请求资源名称及HTTP协议版本
  • 请求头：描述客户端的相关信息及请求的哪台主机
  • 一个空行
  • 请求体

请求行

请求方式：GET、POST、PUT、DELETE、HEAD、OPTIONS、TRACE

GET和POST的区别：GET可以在URL后面加上?的形式传递数据给服务器，多个数据用&分割，数据容量通常不能超过1K，POST方式则可以发送更多的数据

请求资源名称：URL地址

Http协议版本：Http 1.1

请求头

常见请求头

类型 Desc
accept 所支持的数据类型
Accept-Charset 所支持的字符集
Accept-Encoding 支持的压缩格式
Accept-Language 它的语言环境
Host 想访问哪台主机
If-Modified-Since 缓存数据的时间
Referer 客户机是哪个页面来的 防盗链
Connection 请求完后是断开链接还是维持链接

空行

HTTP响应格式

  • 状态行：HTTP版本、状态及状态描述
  • 响应头：描述服务器的基本信息及数据描述，可以通知客户端如何处理回送的数据
  • 一个空行
  • 响应体：服务器向客户端回送的数据

状态行

响应码 描述
100~199 成功接收请求，需要客户端继续提交下一次请求才能完成整个处理过程
200~299 成功接收请求并完成整个处理过程，常用200
300~399 为完成请求，客户端需进一步细化请求，常用302，307和304
400~499 客户端请求由错误，常用404（资源找不到）
500~599 服务器端出现错误，常用500

302：服务器调用另外资源，叫请求转发

307：通知浏览器访问另外资源，叫重定向

403：禁止，服务器拒绝请求

响应头

常见响应头

Header Desc
Location 告诉浏览器跳转到哪里
Server 服务器型号
Content-Encoding 数据压缩格式
Content-Length 回送数据的长度
Content-Language 语言环境
Content-Type 回送数据类型
Refresh 告诉浏览器定时刷新
Content-disposition 告诉浏览器以下载方式打开数据
Transfer-Encoding 数据以分块方式回送
Expires -1,控制浏览器不要缓存

在Servlet中，可以通过设置响应码和Location来告诉浏览器进行跳转，即重定向。

空行

响应体

常用JSON格式传输

处理

中间的处理则是通过Tomcat和Servlet来进行，Tomcat负责连接，Servlet负责处理

SpringMVC，则是通过DispatchServlet，用于转发请求，至于请求的处理交给Controller。
