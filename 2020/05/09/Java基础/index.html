static和final

static关键字

静态方法只能访问静态变量，静态变量都能被方法访问

static在类加载的时候就会进行初始化，而实例变量和实例方法都是在对象初始化时才会产生

子类不能重写父类的静态方法，静态方法是一开始就指定了内存区域，子类的静态方法在初始化的时候也会指定不同的内存区域

final关键字

类的private方法会隐式地被指定为final方法

final修饰成员变量，初始化必须在定义时或构造方法中，之后不能更改。对于引用类型，不能更改指向的对象，但可以更改对象的值。

final修饰局部变量，只要在使用前初始化

final修饰方法，无法被重写

final修饰类，无法被继承

static和final的比较

static：静态变量，类加载过程会进行初始化，不需要等到创建对象

final：常量，无法被修改，

String

String的方法返回String类型时都会返回一个新建的String对象，浪费空间

StringBuffer线程安全是在方法上加了synchronized

抽象类和接口

  • 接口都是抽象方法，抽象类可以有实例方法和构造方法
  • 接口被类实现，抽象类被子类实现
  • 接口提供的是规范，抽象类是抽取子类的公共特性
  • 抽象类是先要有几个子类，经过重构得来的，接口可以预先设计。抽象类自下往上，接口自上往下

instanceof

判断一个对象是否是某种类型，类型包括类、抽象类、接口

自增自减的区别:

放在前面，就是先进行自增操作，再进行操作；放在后面，就是先用变量进行操作，操作完之后再进行自增操作

引用传递还是值传递？

参数都是会复制一份传递的，基本数据类型复制后，不影响原数据，引用类型复制的是引用，引用仍然是指向同一个对象的，所以可以根据复制的引用来改变原有的对象。

X{n}：X恰好出现n次

X{n,}：X最少出现n次

X{n,m}：X最少出现n，最多出现m次

?：出现0次或1次，=> X{0,1}

*：0次或多次，=> X{0,}

+：1次或多次， => X{1,}
