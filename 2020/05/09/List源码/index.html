List源码分析

Java集合中，分为List，Set，Map。

  • List是有序可重复
  • Set是无序不重复
  • Map是键值对

（集合继承结构图，待补充）

(ArrayList继承结构图，待补充)

ArrayList属于List系列，那么ArrayList如何实现这些功能的呢？首先从类定义着手。

ArrayList(Java 7)

定义

  • 父类：AbstractList，包含了一些List的默认实现。
  • 实现的接口：List，RandomAccess, Cloneable, Serializable

ArrayList父类AbstractList

  • 父类：AbstractCollection
  • 实现的接口： List

AbstractList父类AbstractCollection

  • 实现的接口：Collection

顶级接口Collection

  • 继承的接口：Iterable

顶级接口Iterable

  • Iterable接口中，只有一个返回迭代器对象Iterator的方法

顶级接口Iterator

在Iterator接口中，定义了三个方法，也就是Collection中，只要通过Iterable接口iterator方法获取到了Iterator对象，就能对元素进行迭代操作。

Iterable代表有迭代能力，Iterator代表迭代能力的具体实现。这两个接口是集合迭代器的一个实现，真正的集合特性在Collection中定义。接下来就看看Collection接口的定义。

Collection接口

首先Collection接口只继承了Iterable接口，实现了迭代器的功能。其他的则是所有集合具备的基本属性。

接口的方法分成三部分：

  • Collection本身定义的方法
    • 查询操作
    • 修改操作
    • 批量操作
  • Iterable定义的方法，iterator()代表能够进行迭代
  • Object类的方法，equals()和hashCode()代表Collection中的元素能够进行比较并进行Hash

所以Collection的接口就看完了。接下来看Collection接口的默认实现类AbstractCollection

AbstractCollection

Collection接口的默认实现类。

AbstractCollection到此源码就结束了。基本上对集合都有了默认的实现。需要子类去实现的方法就很少了。再看List在Collection接口的基础上增加了哪些特性。

List接口

List接口继承了Collection，所以Collection的基本特性，也就是方法，List接口都有。同时List增加了一些具体的功能，所以Collection的方法就不进行介绍了。

从上面可以看出，List在Collection的基础上增加了位置访问的操作，也就是List的特性。然后我们再来看AbstractList，看看List的默认实现。

AbstractList

从类的定义中可以看出来，AbstractList继承了AbstractCollection，而List也是继承了Collection接口的，所以AbstractList中对Collection的实现比较少，主要就是对List接口的默认实现。

ListItr中，next()方法和previous()方法解析
首先明白两个变量的意义：

  • cursor：Index of element to be returned by subsequent call to next.也就是说，cursor代表后续调用next()方法将返回的索引
  • lastRet：Index of element returned by most recent call to next or previous. Reset to -1 if this element is deleted by a call to remove.也就是说，lastRet是最近访问过的元素位置

ListIterator接口

到这里，AbstractList的代码就完成了。AbstractList也完成List的大部分实现。接下来就是ArrayList具体类了

ArrayList

ArrayList实现的三个接口 RandomAccess, Cloneable, java.io.Serializable都是标记接口，没有方法。所以ArrayList就需要将AbstractList中所有abstract方法进行实现，所有空实现进行重写。
我们可以得出要实现的方法有：

  • size() 在AbstractCollection中，AbstractList中都没有写出
  • get在AbstractList中也没有实现
  • set、add、remove在AbstractList中都是抛出异常

如果数组长度给定了，那么不管有没有实际的数据，length得到的都是固定的长度。
