MySQL索引问题

  1. 表关联查询必须小表驱动大表

  2. 索引失效的情况:

    • where中出现函数
    • 单表查询,相同字段尽量不用OR,用UNION代替
    • LIKE语句用%开头
    • 组合索引没有遵循从左到右

  3. 索引不宜过多,尽量不要超过10个

  4. 必须有主键

Explain怎么看

id：嵌套查询中id越大越先执行

select_type:

type：很重要，连接使用哪种类别，有无用索引。分别由system、const、eq_ref、ref、range、index、all

possible_key：

key：MySQL真正使用的索引

rows：预估扫描的行

MySQL优化

  1. 表关联查询必须小表驱动大表

  2. 索引失效的情况:

    • where中出现函数

    • LIKE语句用%开头

  3. 索引不宜过多,尽量不要超过10个

  4. 必须有主键

Explain优化,能够显示了MySQL如何使用索引

  • select_type: slect类型

  • table: 输出的行所引用的表

  • partitions:表分区

  • type: 联接类型,

  • possible_keys:MySQL能使用哪个索引在该表中找到行(重要)

  • key:MySQL实际使用的索引,没有使用索引则为NULL(重要)

  • key_len:使用索引的长度,key为NULL的话这里也为NULL

  • ref:显示使用的哪个列和索引一起从表中选择行

  • rows: 执行查询要检查的行数

  • filtered : 通过条件过滤出的行数百分百(重要)
