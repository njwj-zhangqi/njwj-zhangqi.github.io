char和varchar的区别

char固定长度，但是浪费空间，varchar可变长度，存储的值是占用的字节加上用来记录长度的字节长度

各种字段的最佳大小：比如用户名等等

解决：用户名和密码采用varchar64

三范式

第一范式：列不可再分

第二范式：在1NF基础上，表必须有主键，且非主键必须依赖于主键，否则容易产生冗余（适当的冗余也是可以的）

第三范式：在2NF基础上，非主键列必须直接依赖于主键，而不能依赖非主键列，再通过非主键列依赖主键

Oracle数据库搭建语句

Oracle数据库的创建，和MySQL创建的步骤不同，MySQL中是直接create database 数据库名；但是Oracle在中间添加了一个表空间，由表空间来连接用户和数据库。所以新建数据库的时候是先创建表空间，指定用户再授权，有了表空间后，会更加灵活。

默认在同一个数据库实例中，新建一个表空间，创建用户后需要指定表空间，也可以再创建一个用户指定相同的表空间。之后如果需要更改某个用户的表空间，也可以通过语句操作。

也就是表空间和用户实际上没有固定的关系，但是用户一般都有一个默认的表空间。建表语句 如果不指定表空间，则在当前用户的默认表空间中创建，指定之后就是在指定表空间中创建

所以Oracle建库语句步骤一般为：

  1. 新建数据表空间和临时表空间
  2. 新建用户并指定表空间
  3. 给用户授权
  4. 切换用户登录
  5. 登录后开始建表

Oracle查询优化相关

  1. Oracle执行的是执行计划，而不是按照SQL语句顺序来执行.
  2. 如果在where子句中有or或者根据索引列之后的列来进行过滤，则不会走索引，而进行全表扫描
  3. 执行的顺序:
    • from –> where –> group by –> order by
    • 执行from子句是从右往左执行，选择记录条数少的表放在右边
    • where子句也是从后向前执行，因此过滤最大条数的子句放在最末尾，对于多表之间的连接，则需要写在前面
    • select 子句尽量避免使用来执行，会自动转换成列名，浪费时间

Oracle索引失效问题

小表的时候不需要建立索引

  1. 使用索引列的Null 和Not Null会失效，如where name is null
  2. 使用函数，如substr(name,1,3)=’ABC’会导致name失效
  3. 进行计算，如 age/ 10> 10会失效，可以改成age >10* 10
  4. 使用Not(!=,<>)会失效,换成其他情况

Oracle效率相关

使用exists替代in，not exists 代替not in，或者外连接也比in好，in 会对子查询的表执行一个全表遍历

Oracle delete和truncate

delete 删除时会记录日志，如果未提交是可以恢复的。删除较慢。 truncate不会记录日志，删除很快。
