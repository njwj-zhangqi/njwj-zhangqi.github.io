SpringMVC的运行流程

  • SpringMVC先将请求发送到DispatchServlet
  • DispatchServlet找到HandlerMapping，通过getHandler()返回一个HandlerExecutionChain，然后HandlerAdapter.support找到处理请求的Controller
  • 通过Adapter.handler()把request和response对象传递到对应的Controller，处理后返回一个ModelAndView
  • 查询ViewResolver视图解析器，找到ModelAndView对象指定的视图对象
  • 视图对象负责渲染返回给客户端

事务

  1. 在BaseDao的泛型类中,没有用@Repository注解.

  2. 在BaseServiceImpl需要BaseDao作为实例变量,但是不能用@Autowired,因为BaseDao有多个子类,不能根据类型自动注入

BaseService里面和BaseDao方法一样, 但是在具体Service里面, 比如list,在dao里面需要Service传递hql过来, 但是在service里面的list()方法就不要Controller传递数据过来,所以自己重载了list方法,或者直接在INameService中直接定义一个重载的方法,实现的时候调用BaseServiceImpl的list实现.

  1. 使用事务来进行数据库的增删改

  2. 在spring里面配置tx,启动事务注解,配置事务管理器

  3. 用@transactional注解声明需要事务的方法(主要用在Service里面)

  4. 事务的隔离级别

  5. 脏读 : 一个事务读取到另一事务未提交的更新数据(没提交数据)

  6. 不可重复读 : 一个事务多次读取时, 能读取到在此过程中另一事务提交的更新数据(应该是不可以读到的).只有重新开启事务才能读取到提交的更新数据

  7. 幻读 : 一个事务能够读取到 另一事务已提交的插入数据
