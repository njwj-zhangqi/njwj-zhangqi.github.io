查看定义

在AbstractAccessDecisionManager中没有decide方法的实现，它有三个子类，分别是

  • AffirmativeBased，Affirmative，肯定的，积极的，agreeing with a statement or to a request.
    • 只要有同意就表示通过
    • 如果全部弃权也表示通过；
    • 如果没有一个人投赞成票，但是有人投反对票，则将抛出AccessDeniedException。
  • ConsensusBased，Consensus，共识，general agreement
    • 如果赞成票多于反对票则表示通过。
    • 反过来，如果反对票多于赞成票则将抛出AccessDeniedException。
    • 票数相同且不等于0，且allowIfEqualGrantedDeniedDecisions的值为true则通过
    • 都弃权，且allowIfAllAbstainDecisions为true则通过
  • UnanimousBased，Unaminous，(of two or more people) fully in agreement，一致的
    • 只要有反对，就拒绝
    • 没有反对有赞成，则通过
    • 全部弃权，且allowIfAllAbstainDecisions为true则通过

RoleVoter

RoleVoter是Spring Security内置的一个AccessDecisionVoter，其会将ConfigAttribute简单的看作是一个角色名称，在投票的时如果拥有该角色即投赞成票。

如果ConfigAttribute是以ROLE_开头的，则将使用RoleVoter进行投票。

  • 当用户拥有的权限中有一个或多个能匹配受保护对象配置的以ROLE_开头的ConfigAttribute时其将投赞成票；
  • 如果用户拥有的权限中没有一个能匹配受保护对象配置的以ROLE_开头的ConfigAttribute，则RoleVoter将投反对票；
  • 如果受保护对象配置的ConfigAttribute中没有以ROLE_开头的，则RoleVoter将弃权。

AuthenticatedVoter

AuthenticatedVoter可以处理的ConfigAttribute有IS_AUTHENTICATED_FULLY、IS_AUTHENTICATED_REMEMBERED和IS_AUTHENTICATED_ANONYMOUSLY。

如果ConfigAttribute不在这三者范围之内，则AuthenticatedVoter将弃权。否则将视ConfigAttribute而定

如果ConfigAttribute为IS_AUTHENTICATED_ANONYMOUSLY，则不管用户是匿名的还是已经认证的都将投赞成票；

如果是IS_AUTHENTICATED_REMEMBERED则仅当用户是由Remember-Me自动登录，或者是通过登录入口进行登录认证时才会投赞成票，否则将投反对票；

而当ConfigAttribute为IS_AUTHENTICATED_FULLY时仅当用户是通过登录入口进行登录的才会投赞成票，否则将投反对票。

@RequestMapping在同一个Controller中，不能完全一样

Ajax参数的说明：

  • url : 请求的URL
  • data ： 发送给服务器的数据
  • type ： 请求类型，有GET，POST等

  • success ：成功返回的回调函数

Controller注解的说明：

  • @RequestMapping，在方法上，URL映射路径，与ajax的url相对应
  • @RequestBody，在参数中，接收针对payload方式传递的参数,常用于JSON格式数据,只接收JSon对象的字符串
  • @PathVariable，接收针对路径中的参数
  • @RequestParam，针对普通的参数传递，不想使用POJO去接收，或者转换参数的名称
  • @ResponseBody，不加此注解，如果返回String，则会被解析为jsp文件，加上后正常返回String

@Cacheable和@CacheEvict注解。
注解放在单个方法上，缓存行为只会运用到这个方法上，如果注解在类级别，缓存行为就会应用到这个类的所有方法。

  • @Cacheable Spring在调用方法之前，首先应该在缓存中查找方法的返回值，能找到就返回缓存的值，否则的话，这个方法就会被调用，返回值会被放到缓存中
  • @CachePut 表明Spring应该将方法的返回值放到缓存中，在方法的调用前并不会检查缓存，方法始终都会被调用
  • @CacheEvict 表明Spring应该在缓存中清除一个或多个条目
  • @Caching 这是一个分组的注解，能够同时应用多个其他的缓存注解

@Cacheable和@CachePut的共有属性：

  • String[] value 要使用的缓存名称
  • String condition SpEL表达式，如果得到的值是false，不会将缓存应用到方法调用上
  • String key SpEL，用来计算自定义的缓存key
  • String unless SpEL，得到的值是true，返回值不会放到缓存之中

可以将缓存注解放在接口上，而不是实现类上，这样所有实现类都会遵循同样的规则。

@CachePut的目的是，新增一条记录后，很可能很快就用到，所以放在缓存中更快速。

  • #root.args 传递给缓存方法的参数，数组
  • #root.caches 该方法执行时所对应的缓存，数组
  • #root.target 目标对象
  • #root.targetClass 目标对象的类
  • #root.method 缓存方法
  • #root.methodName 缓存方法名
  • #result 方法调用的返回值，不能用在@Cacheable注解上
  • #Argument 任意方法的参数名(#argName) 或参数索引(#a0)

condition和unless属性，如果condition的计算结果为false，不会去缓存查，也不会放入缓存。如果unless计算结果为true，返回的数据不会放到缓存中。

unless只有在有返回值情况下才发挥作用，而condition在进入方法前就需要进行判断

@CacheEvict主要使用在删除数据的情况下，移除缓存

@CacheEvict的属性

  • String[] value
  • String key
  • String condition
  • boolean allEntries 为true，特定缓存的所有条目都会被移除
  • boolean beforeInvocation 为true，在方法调用之前移除条目，为false(默认值),在方法调用之后再移除条目

tomcat原理

Tomcat服务器的启动是基于一个server.xml文件的，Tomcat启动的时候首先会启动一个Server，Server里面就会启动Service，Service里面就会启动多个”Connector(连接器)”，每一个连接器都在等待客户机的连接，当有用户使用浏览器去访问服务器上面的web资源时，首先是连接到Connector(连接器)，Connector(连接器)是不处理用户的请求的，而是将用户的请求交给一个Engine(引擎)去处理，Engine(引擎)接收到请求后就会解析用户想要访问的Host，然后将请求交给相应的Host，Host收到请求后就会解析出用户想要访问这个Host下面的哪一个Web应用,一个web应用对应一个Context。

进程就是一个程序，最小调度单元。

进程的状态：

  1. 新建：分配进程id，优先级等
  2. 就绪：一旦新建完毕，被放在就绪队列，等待CPU选择执行
  3. 运行：在CPU中运行
  4. 阻塞：可能要进行IO操作，放到阻塞队列中，如果阻塞完成，就放回到就绪队列，等待CPU调用

ThreadPoolExecutor

ThreadPoolExecutor执行execute方法有4种情况：
1）如果当前运行的线程少于corePoolSize，则创建新的线程来执行任务。
2）如果运行的线程等于或者多余corePoolSize，则将任务加入BlockingQueue中，在等待队列中，等待有新的线程可以运行。
3）如果BlockingQueue队列满了，且没有超过maxPoolSize，则创建新的线程来处理任务。
4）如果创建的线程超过maxPoolSize，任务会拒绝，并调用RejectExecutionHandler.rejectedExecution()方法。
